## 2. “离上架还差多远？” (The Gaps)

虽然核心逻辑已通，但要达到“成熟上架”标准，还需要跨越以下 **3 个坎**：

### A. 后端数据一致性 (Backend Polish)
*   **Bug 隐患**：代码中图片上传指向 `campus` 存储桶，但 SQL 配置里叫 `posts`。
*   **数据隔离**：目前是 Demo Mode 运行，需要确保 Supabase 的 RLS (行级安全策略) 完美适配所有同学的真实登录。

### B. Agent 的鲁棒性 (Agent Reliability)
*   **痛点**：WebView 自动化极度依赖 HKBU 官网的 HTML 结构。官网一旦改版，Agent 就会失效。
*   **建议**：增加更完善的错误提示（例如：“由于官网架构调整，请手动在下方 WebView 操作”），并建立一套云端更新 DOM 脚本的机制。

### C. 生产环境合规 (Compliance & Production)
*   **隐私政策**：涉及用户登录和学校数据，上架必须有 Privacy Policy。
*   **视觉素材**：缺一套精美的 App Icon 和启动图 (Splash Screen)。
