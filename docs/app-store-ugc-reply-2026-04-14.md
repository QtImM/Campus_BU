We have reviewed the current build of HKCampus and confirmed that the app includes the required precautions for user-generated content moderation.

1. Terms of Use / EULA before accessing user-generated content

Before a user can access community content, the app presents a Community Safety / EULA agreement that must be accepted first. This agreement clearly states that HKCampus has zero tolerance for objectionable content and abusive behavior, and it explains prohibited content, user moderation, and enforcement actions.

2. Objectionable content filtering

The app includes objectionable content filtering for user-generated content. In the current implementation, content safety checks are applied to community content types including posts, comments, forum content, reviews, course exchange content, teaming content, and messages. In addition, server-side moderation rules reject disallowed abusive, sexual, hateful, violent, harassing, or spam-like content before it is published.

3. Reporting objectionable content

Users can report objectionable content from the in-app content action menus. The reporting flow allows the user to choose a reason and submit the report for review. Reports are recorded in the moderation workflow for follow-up handling by the developer.

4. Blocking abusive users

Users can block abusive users from the same in-app action menus. After blocking, the blocked user's content is removed from the current user's visible feed immediately. Blocking also creates a moderation report for developer review, and blocked user relationships are enforced so abusive users cannot continue interacting normally with the blocking user.

5. Developer action within 24 hours

The moderation workflow includes report tracking and a 24-hour response target. Reported content can be reviewed in the moderation workbench, where the developer can remove reported content, ban abusive users, or both. Once a user is banned, server-side enforcement prevents that user from creating new user-generated content.
