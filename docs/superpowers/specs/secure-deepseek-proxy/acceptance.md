# Acceptance

## 功能验收标准
1. 已登录用户可正常使用 Agent 与翻译能力。
2. 未登录用户调用代理时返回 401/403，客户端能提示登录或无权限。
3. 客户端不再出现任何 DeepSeek Bearer Key。
4. 代理接口错误不会导致应用崩溃，且有可读提示。

## 安全验收标准
1. 打包产物与源码中不再依赖 EXPO_PUBLIC_DEEPSEEK_API_KEY。
2. DeepSeek Key 仅存在于服务端环境变量 DEEPSEEK_API_KEY。
3. 代理函数拒绝非法模型名与超限参数。
4. 代理函数对错误响应做脱敏处理，不回传上游完整敏感报文。

## 边界条件
1. DeepSeek 上游超时：客户端收到超时提示，支持重试。
2. Supabase JWT 过期：客户端收到 401 并引导重新登录。
3. 空消息或超长消息：代理返回 400，客户端提示输入问题。
4. 并发高峰触发限流：返回 429，客户端提示稍后重试。

## 回归检查点
1. 现有 Agent 对话主流程可继续工作。
2. summarizeHistory 与 memory_extractor 相关链路仍能获取模型输出。
3. 翻译按钮展示逻辑正确，目标语言切换仍可用。
4. 现有 Supabase 业务查询不受影响。

## 失败判定条件
1. 任一客户端路径仍直接请求 DeepSeek 官方地址并携带 Bearer Key。
2. 未登录用户可调用代理成功。
3. 登录用户核心功能失败率显著上升且无降级提示。
4. 单测或手工验证发现关键回归未闭环。

## 人工验证步骤
1. 登录账号后进入 Agent 页面，发送问题并确认得到正常回复。
2. 登录账号后在可翻译文本处点击翻译，确认返回翻译结果。
3. 退出登录后重试上述动作，确认出现鉴权失败提示。
4. 抓包或日志检查客户端请求，确认未包含 DeepSeek API Key。

## 自动化测试建议
1. 单测：proxy_client 参数校验与错误映射。
2. 单测：llm.ts 在代理成功/失败/超时场景下行为。
3. 单测：translate.ts 在代理成功/失败场景下行为。
4. 集成测试：模拟已登录与未登录状态调用代理。
